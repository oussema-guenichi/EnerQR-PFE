🐍 Backend — Python / FastAPI enerqr-backend/
FichierTypeRôle & Description
main.py Core Cœur de l'API REST. Contient tous les endpoints FastAPI : authentification (/auth/login, /auth/register), gestion des clients, techniciens, anomalies, garanties, factures et le routing IA.
models.py Base de Données Définit les modèles SQLAlchemy (ORM). Tables : Users, Pumps, Anomaly, Garantie, Facture, ScanLog. Chaque classe Python correspond à une table SQL automatiquement créée.
database.py Base de Données Configuration du moteur SQLite. Crée la session de base de données et le binding engine. Fournit la dépendance get_db() pour les endpoints FastAPI.
schemas.py Core Schémas Pydantic pour la validation des données entrantes et sortantes. Définit UserCreate, UserResponse, StatsResponse, AnomalyCreate, FactureUpdate, etc.
migrate.py Base de Données Script de migration manuelle de la base de données. Ajoute ou modifie des colonnes dans les tables existantes sans perdre les données (ALTER TABLE).
seed_db.py Base de Données Script de peuplement de la base de données avec des données de test (clients fictifs, pompes, anomalies) pour faciliter les démonstrations.
check_db.py Configuration Script utilitaire pour inspecter et vérifier le contenu de la base de données SQLite directement depuis le terminal.
requirements.txt Configuration Liste des dépendances Python : FastAPI, SQLAlchemy, Pydantic, NumPy, Scikit-learn, Uvicorn, HTTPX. Utilisé par pip install -r requirements.txt.
test_ollama.py IA Script de test pour vérifier la connexion avec le modèle Ollama/Mistral local. Envoie un message de test et affiche la réponse du LLM.
enerqr_v3.db Base de Données Fichier SQLite principal de l'application (base de données active). Contient toutes les données réelles : utilisateurs, pompes, anomalies, garanties et factures.
🤖 Modules IA — enerqr_ai/ enerqr-backend/enerqr_ai/
FichierTypeRôle & Description
prediction.py IA Algorithme de prédiction de consommation hebdomadaire (S+1 à S+4). Prend en compte la zone climatique du client (H1/H2/H3) et la saison actuelle pour calculer un multiplicateur saisonnier réaliste.
anomaly.py IA Algorithme de détection d'anomalies (surconsommation). Compare conso_avant vs conso_après avec un ratio de décision (seuil 0.8). Génère automatiquement une alerte si anomalie détectée.
chat.py IA Module chatbot intelligent intégrant Ollama (modèle Mistral local). Interprète les questions du client sur sa pompe à chaleur et génère des réponses contextualisées.
⚛️ Frontend Web — React.js / Vite enerqr-frontend/src/
FichierTypeRôle & Description
App.jsx Routing Routeur principal de l'application (React Router DOM). Définit toutes les routes : /, /auth, /admin, /dashboard/:id, /activate/:id, /scan. Point d'entrée de toute la navigation.
api.js Core Fichier de configuration centralisée de l'URL du backend (API_BASE_URL). Permet de changer l'adresse du serveur en un seul endroit pour toute l'application.
AuthPage.jsx UI Page d'authentification unifiée (Connexion + Inscription). Design Glassmorphism. Gère les 3 rôles (Client, Technicien, Admin) avec champs conditionnels et routing intelligent post-login.
AdminDashboard.jsx UI Dashboard principal pour Admin et Technicien. Interface CRUD complète : gestion clients, pompes, anomalies (3 niveaux), garanties, factures, techniciens. Graphiques Recharts et génération QR Code.
DashboardClient.jsx UI Espace personnel du client. Affiche KPIs (économies, consommation), graphique IA de prédiction 4 semaines (Recharts AreaChart), technicien responsable et chatbot Ollama intégré.
ActivationPage.jsx UI Page d'activation RGPD. Affichée lors du premier scan QR ou après inscription. Le client doit accepter les conditions de confidentialité avant d'accéder à son dashboard.
ScanRedirect.jsx Routing Page de redirection intelligente après scan QR. Interroge l'API pour identifier la pompe et son propriétaire, puis redirige vers l'activation (si premier scan) ou le dashboard client.
HomePage.jsx UI Page d'accueil publique du site. Présente le projet EnerQR, les fonctionnalités clés et le call-to-action vers l'authentification.
App.css UI Styles globaux de l'application : système de design tokens (variables CSS), effets Glassmorphism, animations fade-in, composants glass-card, dark/light mode.
index.css UI Reset CSS et styles de base. Définit les polices Google Fonts (Inter, Outfit), les variables de couleurs primaires et les styles des éléments HTML fondamentaux.
📱 Application Mobile — Flutter / Dart enerqr_mobile/lib/
FichierTypeRôle & Description
main.dart Core Point d'entrée de l'application Flutter. Configure le MaterialApp, le thème global et lance le LoginScreen comme écran initial.
api/api_service.dart Core Service API centralisé. Contient toutes les fonctions HTTP (login, register, getStats, getAIPrediction, getAllClients, etc.). Configure l'URL du backend (IP configurable selon réseau WiFi).
screens/login.dart UI Écran d'authentification mobile (Connexion + Inscription). Design dark Glassmorphism avec sélecteur de rôle. Routing intelligent post-login avec gestion du consentement RGPD.
screens/unified_dashboard.dart UI Dashboard unifié multi-rôles. Affiche dynamiquement l'interface Admin, Technicien ou Client selon le rôle connecté. Inclut statistiques réelles API, KPIs client, prédiction IA et gestion anomalies.
screens/dashboard.dart UI Dashboard client simplifié (version initiale). Affiche les économies réalisées et les informations de base de la pompe à chaleur.
screens/technician_dashboard.dart UI Dashboard spécialisé technicien (version initiale). Permet de voir et gérer les clients assignés, modifier les données de consommation et consulter les anomalies détectées.
